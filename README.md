# Лабораторная работа №12. Жизненный цикл Activity (Activity Lifecycle)
## Название проекта
"Grade Clicker" — интерактивный кликер для набора
баллов по курсу "Разработка мобильных приложений".
## Цель работы
Изучить жизненный цикл Activity в Android и научиться правильно
обрабатывать переходы между состояниями. В результате лабораторной работы будет
создано интерактивное приложение "Grade Clicker" (Кликер оценок), которое
демонстрирует работу lifecycle callbacks и сохранение состояния при изменении
конфигурации устройства.
## Контрольные вопросы
1. При запуске вызываются три метода по порядку:
- onCreate() — создание Activity
- onStart() — Activity становится видимой
- onResume() — Activity получает фокус (активна)
2. Современные версии Android агрессивнее управляют lifecycle, поэтому различие между onPause() и onStop() стало менее очевидным в повседневном использовании.
3. onRestart() — вызывается только при перезапуске после Stop
4.  Поворот экрана вызывает configuration change (изменение конфигурации)
5. 
remember:
- Сохраняет значение только при рекомпозиции
- Теряет значение при пересоздании Activity
   
rememberSaveable:
- Сохраняет значение при рекомпозиции
- И ПРИ configuration changes (поворот, смена языка и т.д.)
6.
- Вызывается один раз при создании Activity
- Здесь выполняется инициализация: установка UI через setContent()
- Необходимо вызвать super.onCreate() для завершения создания Activity
7.
- Log.d() — записывает debug-сообщение в Logcat
- Первый параметр — тег (TAG)
- Второй параметр — сообщение ("onCreate Called")
8. OnCreate()
